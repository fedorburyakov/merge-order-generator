# Генератор заказов для merge-игры

Небольшой прототип системы генерации заказов для мобильной **merge-2 игры**

[Описание тестового задания (Notion)](https://www.notion.so/kukusik-ept/Game-Designer-31466ecf271980d78703fcd09d689052?source=copy_link)

Скрипт имитирует работу системы, которая формирует новые заказы игроку на основе:

* текущего пула активных заказов
* используемых линеек предметов
* уровня игрока
* сложности предметов
* ограничений системы (лимиты high-предметов, сложность пула и т.д.)

Основная цель системы — **поддерживать баланс сложности и разнообразие заказов**, избегая повторений и перегрузки игрока слишком сложными заданиями.

---

# Структура проекта

```
.
├── generator.py
├── config.json
├── lines.json
├── orders.json
└── README.md
```

## generator.py

Основной скрипт генерации заказов

Функционал скрипта:

* читает данные из `config.json`, `lines.json` и `orders.json`
* анализирует текущий пул активных заказов
* считает суммарную сложность пула
* выбирает линейки предметов с учетом весов и ограничений
* выбирает тир предмета внутри линейки
* формирует новый заказ (1–3 предмета)
* проверяет ограничения системы (сложность, лимиты по линейкам и high-предметам)
* добавляет новый заказ в пул

---

## config.json

Файл конфигурации системы

Содержит основные параметры балансировки:

* максимальное количество активных заказов
* лимиты предметов одной линейки
* лимиты сложных заказов
* коэффициенты влияния уровня игрока
* параметры выбора тиров
* ограничения суммарной сложности пула

Этот файл используется для **балансировки системы без изменения кода**

---

## lines.json

Описание всех доступных **линеек предметов**

Для каждой линейки задаются:

* тип генератора
* шанс выпадения предметов линейки
* веса выбора тиров
* список доступных тиров
* порог high-предметов
* дополнительные множители веса (например для дополнительных линеек)

Этот файл определяет **структуру игровых предметов и генераторов**

---

## orders.json

Файл с текущими **активными заказами**

Каждый заказ содержит:

* `OrderID`
* список предметов
* рассчитанную сложность заказа

Скрипт использует этот файл для анализа текущего состояния системы и добавления новых заказов

---

# Принцип работы генератора

Система генерации работает следующим образом:

1. Анализирует текущий пул заказов:

   * какие линейки уже используются
   * есть ли high-предметы
   * какая суммарная сложность заказов

2. Рассчитывает текущую нагрузку системы

3. Выбирает линейку предметов с учетом:

   * частоты использования
   * наличия high-предметов
   * базовых весов генераторов

4. Выбирает тир предмета внутри линейки

5. Формирует новый заказ из 1–3 предметов

6. Проверяет ограничения системы:

   * лимиты по линейкам
   * лимиты сложных заказов
   * допустимую суммарную сложность пула

7. Если условия выполняются — заказ добавляется в пул

# Запуск скрипта

Требования:

* Python 3.10 или новее

Запуск:

```
python generator.py
```

После запуска скрипт:

* анализирует текущие заказы
* генерирует новый заказ
* обновляет файл `orders.json`.

# Назначение прототипа

Этот проект демонстрирует системный подход к генерации заказов в merge-играх:

* контроль сложности через модель вычисления сложности
* балансировка вероятностей выбора предметов
* предотвращение повторений
* адаптация системы к прогрессу игрока

Прототип может использоваться как база для дальнейшего развития системы экономики и генерации контента

---

# Ограничения прототипа

Данный прототип моделирует только **логику генерации заказов** и не учитывает ряд игровых систем:

* состояние игрового merge-поля
* специальные механики из раздела **«Дополнительно»** (например, *Rave Reviews*, *Frozen Items*, *Bubbles*)
* реальные игровые сценарии начальных уровней

В реальной игре стартовые уровни обычно имеют **заранее настроенные заказы и параметры**, поэтому поведение системы на ранних этапах прогрессии может отличаться от результатов данного прототипа
